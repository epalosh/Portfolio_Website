Most Comprehensive Personal Project

Here are some key features I implemented:

  • Secure User Authentication: A robust system to ensure safe and private user access.
  • User Account Management: Users can link their data to their account, modify details, and delete accounts as needed.
  • Real-Time Data Updates: A custom "publisher" infrastructure enables live data updates in the UI without requiring a refresh.
  • Dynamic User Interface: All UI components are fully functional, including dynamic displays, buttons, and interactive views.
  • Custom Messaging System: A unique design for messaging and notifications to enhance user communication.
  • Account Interaction Features: Users can add and manage connections, providing a seamless experience.
  • Push Notifications: Integrated JavaScript push-notification functionality running on Google Cloud for global user engagement.
  • Global User Statistics: Real-time tracking and display of user data across the globe.
  • In-App Purchases: Full integration of in-app purchases, supporting app revenue generation.
  • LLM-Driven Core Features: Integrated Large Language Model (LLM) support for enhancing core app functionality.
  • OpenAI Integration: Dynamic data provided by OpenAI's API to support intelligent app features.
  • Remote Config: Select data is loaded from the database, and used for setting a global minimum app version across all users.

Functional & Intuitive UI

All UI elements are defined in code and fully customized. I designed the dashboard to immediately display important information, reducing clicks and making the user experience seamless. I went through many iterations of each view, and after several months of feedback, I arrived at the current design!

  • Dashboard: The primary view of the app, which is displayed on app launch. The dashboard shows a horizontal scroll view of message cards, which are ranked by latest activity. If new comments come in, they are displayed. Otherwise messages are shown.

  • Sending Messages: Users are guided through a unique messaging flow that utilizes OpenAI's API to generate questions and prompt users to send a detailed story in their message.

  • Design: Designed using SwiftUI for a uniform appearance across Apple devices with different screen sizes. Additionally, the app looks natural and native to the device.

Authentication & Database

User accounts, user data, and global statistics are fully built-out using Google Firebase. The app interfaces with Firebase through network calls (similar to a REST API) to place data into secure locations based on the user that's logged in.

  • Accounts: Each user can create an account using email & password. Accounts can be labeled by "student", "parent", and "other" so I, the developer, can keep track of who uses the app the most. Users can alse request a password change through a secure email, and make modifications to their account details inside the app (such as updating their name or school).

  • Database: User data is keyed by the user's UID, and secure from other users. Only users that are authorized to access certain areas of the database can do so. Additionally, I designed database queries to be effiecient as possible, as Firebase charges developers based on the number of queries. Therefore, the app is built to scale!

  • Statistics: Global "sent messages" count is stored in the database and updated properly to prevent race conditions.

Google Cloud Functions

Several Javascript functions are running 24/7 on Google Cloud servers to securely monitor database activity. They are authorized for database access, and are functionally "admins" to the application.

  • When a notification is requested (new message/connection/comment), these functions securely execute the request and send a push-notification to the user!
